# PV Forecasting Pre-Submission Review

Review date: 2026-05-23

## Result

The task has a runnable technical closed loop after the difficulty pass. The hidden evaluation tail is longer, the scorer reports short/mid/long horizon errors, ramp and extreme-ramp errors, sunlight-transition error, station-bias error, and daily-energy error. The scoring anchors no longer make an ordinary single HistGradientBoosting model full score.

This confirms the scorer is deterministic, automatic, and has a non-saturating improvement range. The scoring scale was recalibrated for the 2026-05-22 acceptance rule that a real 30min agent run should stay at or below 15 points and a real 2h run should show progress but remain at or below 30 points. A simple HGB model anchors at 2 points; capacity normalization anchors at 5 points; an observed 16min HGB ensemble run anchors at 7 points. A real Harness run on 2026-05-23 reached weighted error `129.706352` and score `30.822215` after about 1h under the previous curve, so that performance level is now an explicit 15-point anchor. The 30-point ceiling is moved to weighted error `112.000000`, and the full-score target remains reserved for deeper feature discovery, model ensembling, ramp-event specialization, daily-energy calibration, station-bias control, and long-horizon calibration.

Real Harness acceptance evidence after the recalibration:

- 30min run `pv_agent_30m_20260523_0331`: timed out at `1800.1332714557648` seconds, `7` rounds, best score `3.4332`.
- 2h run `pv_agent_2h_newkey_20260523_1226`: timed out at `7200.029923439026` seconds, `81` rounds, `57` agent submissions, final best score `14.887849`; the highest submission report observed was `14.938906` with `weighted_error=129.841525`.

This satisfies the intended difficulty shape: the 2h Agent run improves over 30min but does not reach 30 points, and the score continues to have room for stronger long-horizon, ramp-event, station-bias, and ensemble work.

## Isolated Red-Team Check

An isolated worker was given only a copy of `agent-start` under `/tmp/pv-redteam-agent-start`. It did not read the scorer, hidden eval labels, baseline submissions, task config, docs, or split manifest. It ran three visible-data iterations:

- Iter1 weak baseline: public dev MAE `109.1780`, RMSE `251.3921`
- Iter2 statistical model: public dev MAE `122.9077`, RMSE `275.9439`
- Iter3 ExtraTrees-style ensemble: public dev MAE `71.1314`, RMSE `170.4574`

Controller-side hidden scoring of the final isolated submission from the earlier scale returned weighted error `212.081945`; under the recalibrated acceptance scale this maps to a low double-digit score. This is the intended behavior: an autonomous agent can make visible, measurable progress from feature/model work, but a short isolated run does not approach full score.
