# PV Forecasting Data Audit

Audit date: 2026-05-20

## 2026-06-06 Leakage Audit Update

A later audit found a structural leakage risk in the current split: raw `Lxx`
station variants `v1/v2/v3` are encoded as sibling `LocationCode` values and
have deterministic 21-day timestamp offsets. Because the current split treats
each `LocationCode` independently instead of grouping by physical station
family, all hidden-eval rows can be matched to public sibling labels by 21-day
or 42-day shifts.

The repaired dataset removes the shifted variants at the source loader: only
raw `site_v1` files are loaded, while `site_v2` and `site_v3` are excluded.
Primary and additional `site_v1` fragments for the same `LocationCode` are then
merged with deterministic key-level averaging. Hidden eval locations are now
fully held out from public train labels; public dev locations retain a time-tail
split for local iteration feedback. See
`docs/leakage_audit_20260606.md` for the original evidence and scoring impact.

## Dataset Used For Closed-Loop Test

Closed-loop verification used the prepared multi-site PV forecasting dataset packaged in this task directory. The records are field-observation style operating data after deterministic cleaning and station/time de-identification. The agent-facing files intentionally expose only task inputs, labels for public development feedback, and the starter code needed to run the benchmark.

## Integrity Checks

- Canonical CSV files loaded: 24 raw `site_v1` files
- Shifted CSV files excluded: all raw `site_v2` and `site_v3` files
- Raw canonical rows loaded: 1,374,201
- Cleaned rows after global key-level de-duplication: 1,372,483
- Duplicate `LocationCode + DateTime` rows removed by averaging numeric fields: 2,545
- Unique locations after cleaning: 17
- Date range: 2025-01-01 to 2025-08-27
- Required fields present: `LocationCode`, `DateTime`, `WindSpeed(m/s)`, `Pressure(hpa)`, `Temperature(°C)`, `Humidity(%)`, `Sunlight(Lux)`, `Power(mW)`
- Missing numeric cells: none observed in pre-check
- Negative power or sunlight values: none observed in pre-check

The largest duplicate-key issue affects one station, with 825 duplicate keys in
the canonical source. Additional overlaps appear where primary and additional
`site_v1` fragments share a `LocationCode + DateTime` key. Duplicates are not
identical rows, so the split tool uses deterministic key-level averaging both
within each CSV and globally after concatenation.

## Split

Generated by `tools/prepare_pv_benchmark.py`.

- Training rows visible to agent: 1,012,131
- Public dev rows: 96,839
- Hidden eval rows: 263,513
- Public dev locations: 3
- Hidden eval locations: 3

After shifted-variant removal, hidden eval is location-heldout: eval stations do
not appear in public train labels or public dev labels. Public-dev stations keep
the later 40% time tail withheld for agent-visible feedback. This measures
unseen-station generalization while still preserving a realistic public
iteration loop. Hidden eval labels are written only under
`scorer/eval-data/eval_labels.csv`.

Current leakage audit result:

- Public variants: `[1]`
- Eval variants: `[1]`
- Public/eval location overlap: none
- Non-canonical public/eval locations: none
- Same-family shifted temporal proxy hits at `+/-21` or `+/-42` days: `0`

## Data Handling Boundary

The packaged benchmark directory has been scrubbed of raw path markers and reconstruction hints. Provenance evidence should be maintained outside the agent-visible task package by the project owner.
