# PV Forecasting Data Audit

Audit date: 2026-05-20

## Dataset Used For Closed-Loop Test

Closed-loop verification used the prepared multi-site PV forecasting dataset packaged in this task directory. The records are field-observation style operating data after deterministic cleaning and station/time de-identification. The agent-facing files intentionally expose only task inputs, labels for public development feedback, and the starter code needed to run the benchmark.

## Integrity Checks

- CSV files inspected: 72
- Raw rows: 4,125,084
- Cleaned rows after global key-level de-duplication: 4,117,449
- Duplicate `LocationCode + DateTime` rows removed by averaging numeric fields: 7,635
- Unique locations after cleaning: 51
- Date range: 2025-01-01 to 2025-10-08
- Required fields present: `LocationCode`, `DateTime`, `WindSpeed(m/s)`, `Pressure(hpa)`, `Temperature(°C)`, `Humidity(%)`, `Sunlight(Lux)`, `Power(mW)`
- Missing numeric cells: none observed in pre-check
- Negative power or sunlight values: none observed in pre-check

The largest duplicate-key issue affected one station family, with 825 duplicate keys per variant. Additional overlaps appeared across source files after station/time de-identification. Duplicates were not identical rows, so the split tool uses deterministic key-level averaging both within each CSV and globally after concatenation.

## Split

Generated by `tools/prepare_pv_benchmark.py`.

- Training rows visible to agent: 3,311,762
- Public dev rows: 326,138
- Hidden eval rows: 479,549
- Public dev locations: 10
- Hidden eval locations: 10

The split combines location-level holdout and a longer time-tail holdout. For hidden-eval stations, the later 60% of each station's timeline is withheld; for public-dev stations, the later 40% is withheld. The longer hidden tail supports short/mid/long-horizon scoring while still leaving earlier same-station history for learnable calibration. Hidden eval labels are written only under `scorer/eval-data/eval_labels.csv`.

## Data Handling Boundary

The packaged benchmark directory has been scrubbed of raw path markers and reconstruction hints. Provenance evidence should be maintained outside the agent-visible task package by the project owner.
